<h2 id="前言">前言</h2>
<p>最近使用在使用 Cloudflare Pages 部署静态网页的过程中，不小心开启了 Access-policy，导致访问部署项目的历史版本都需要经过验证，但想进入关闭 Access-policy 的页面，必须要先绑定了信用卡。</p>
<p>这次属实是被 Cloudflare 给恶心到了，还好找到了解决方法，也就是通过调用 API 的方式去关闭这个功能。</p>
<h2 id="必要条件">必要条件</h2>
<ul>
<li>熟悉 HTTP 协议接口的使用</li>
<li>熟悉 curl 的操作方式</li>
</ul>
<h2 id="操作流程">操作流程</h2>
<p>根据 Cloudflare API 的文档，<a href="https://api.cloudflare.com/#access-applications-list-access-applications" rel="noreferrer" target="_blank">获取 List Access Applications</a>。</p>
<pre class="shiki shiki-themes vitesse-light vitesse-dark" style="background-color:#ffffff;--shiki-dark-bg:#121212;color:#393a34;--shiki-dark:#dbd7caee" tabindex="0"><code><span class="line"><span>GET accounts/:identifier/access/apps</span></span>
<span class="line"><span></span></span></code></pre>
<p>官方示例：</p>
<pre class="shiki shiki-themes vitesse-light vitesse-dark" style="background-color:#ffffff;--shiki-dark-bg:#121212;color:#393a34;--shiki-dark:#dbd7caee" tabindex="0"><code><span class="line"><span style="color:#59873A;--shiki-dark:#80A665">curl</span><span style="color:#A65E2B;--shiki-dark:#C99076"> -X</span><span style="color:#B56959;--shiki-dark:#C98A7D"> GET</span><span style="color:#B5695999;--shiki-dark:#C98A7D99"> "</span><span style="color:#B56959;--shiki-dark:#C98A7D">https://api.cloudflare.com/client/v4/accounts/699d98642c564d2e855e9661899b7252/access/apps</span><span style="color:#B5695999;--shiki-dark:#C98A7D99">"</span><span style="color:#A65E2B;--shiki-dark:#C99076"> \</span></span>

<p>对 API 请求数据很简单，X-Auth-Email 很明显是我们账户绑定的邮箱，但是 url 参数中的 identifier 与 X-Auth-Key 的值应该如何获得？</p>
<h3 id="获取自己的-identifier">获取自己的 identifier</h3>
<p>登录 Cloudflare 之后，我们可以在主页的 URL 后看到一个看上去像是 ID 的字符串（abcdef1145141919810abcdefabcdefa），没错，这个就是我们的 identifier 。</p>
<p><img src="/images/posts/disable-cloudflare-access-policy/api-tokens.png" alt="Cloudflare 主页"></p>
<h3 id="获取自己的-x-auth-key">获取自己的 X-Auth-Key</h3>
<p>登录 Cloudflare 可在这个页面 <a href="https://dash.cloudflare.com/profile/api-tokens" rel="noreferrer" target="_blank">https://dash.cloudflare.com/profile/api-tokens</a> 的 Global API Key 选项中查看。</p>
<p><img src="/images/posts/disable-cloudflare-access-policy/api-tokens.png" alt="API Tokens 页"></p>

<h3 id="关闭-access-policy">关闭 Access policy</h3>
<p>有了 Access policy 的 ID 之后，就可以使用<a href="https://api.cloudflare.com/#access-applications-delete-access-application" rel="noreferrer" target="_blank">删除 Access Application 的接口</a>关闭 Access policy。Cloudflare 是使用 RESTful 规范设计的 API，删除接口的 URL 就是将上文提到的接口，将 GET 换成 DELETE，并在最后加上 ID。</p>

<span class="line"><span></span></span></code></pre>
