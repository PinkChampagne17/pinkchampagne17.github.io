<h2 id="windows-terminal">Windows Terminal</h2>
<p><a href="https://docs.microsoft.com/en-us/windows/terminal/" rel="noreferrer" target="_blank">Document</a></p>
<p>作为一个开发者避免不了需要使用终端的场景，而 Windows 以往默认的终端不好用也不好看。</p>
<p>现在你可以使用来自微软开发的新终端 Windows Terminal，它的界面已经得到了翻新，并且支持了许多以往没有的配置项。</p>
<p>新版本的 Windows 已经内置了 Windows Terminal 在系统中，你可以检查你的系统中是否已经安装，你可以通过 Microsoft Store 获取。</p>
<h2 id="winget">winget</h2>
<p><a href="https://docs.microsoft.com/zh-cn/windows/package-manager/winget/" rel="noreferrer" target="_blank">Document</a></p>
<p>Microsoft 官方提供的包管理器。</p>
<blockquote>
<p>这个工具的本质其实是从官网下载安装引导程序，自动地帮你点击下一步完成安装。这可能会导致你在安装某些软件后，还自动地帮你安装了全家桶。</p>
</blockquote>
<p>当你不想要遵循引导程序中的默认设置时，请使用 <code>-i</code> 选项以使用交互式安装。</p>
<pre><code>winget install -i vscode
</code></pre>
<p>比如你可能需要在安装 VS Code 时，通过在安装引导程序内的选择帮你自动添加环境变量。或者是你在安装某些软件时，你需要取消勾选 checkbox 以避免安装全家桶。</p>
<h2 id="git-bash">Git Bash</h2>
<p>在安装 Windows 版的 Git 通常都会顺带安装 Git Bash，它使得我们可以在 Windows 下使用和 Linux 一样的命令，我们也可以在 Windows Terminal 中使用 Git Bash。</p>
<pre><code class="language-bash"># 创建一个新文件

# PowerShell
$ ni index.html

# Git Bash
$ touch index.html
</code></pre>
<h2 id="wsl">WSL</h2>
<p><a href="https://docs.microsoft.com/zh-cn/windows/wsl/" rel="noreferrer" target="_blank">Document</a></p>
<p><del>Windows 是最好用的 Linux 发行版之一。</del></p>
<p>WSL 是一种在 Windows 中运行 Linux 的方案，新的 WSL2 本质上是运行在 Windows 内置的虚拟机 Hyper-V 之上，但与 Windows 做了一些集成，方便与 Windows 共同协作。</p>
<p>虽然 WSL 与真正的 Linux 还是有些许差异，但已足够满足大部分的开发场景。</p>
<blockquote>
<p>通常 WSL 是以终端的形式出现，但也可以运行图形化界面的应用。</p>
</blockquote>
<blockquote>
<p>如果你在使用 WSL 时遇到了一些问题，你可以到我的<a href="./%E4%BD%BF%E7%94%A8WSL2%E4%BD%9C%E4%B8%BA%E5%BC%80%E5%8F%91%E7%8E%AF%E5%A2%83%E7%9A%84%E9%85%8D%E7%BD%AE%E8%AE%B0%E5%BD%95">另一篇文章</a>看看有没有你需要的答案。</p>
</blockquote>
<p>目前由官方支持的发行版有：Ubuntu、Debian、SUSE、Oracle Linux 和 Kali 等，它们都可以被很方便地从 Microsoft Store 获取。</p>
<p>你可以在 GitHub 上找到类似 <a href="https://github.com/yuk7/ArchWSL" rel="noreferrer" target="_blank">ArchWSL</a> 之类的仓库，帮助你将你偏好的发行版运行在 WSL 上。</p>
<h3 id="hyper-v-的兼容问题">Hyper-V 的兼容问题</h3>
<p>WSL2 是运行在 Hyper-V 之上的，而 Hyper-V 与许多的虚拟机方案并不兼容。</p>
<p>如果你正在使用 VMware Workstation 或 VirtualBox，它们早在之前发布的版本就兼容了 Hyper-V。如果你使用的不是比较旧的版本，你无需担心。</p>
<p><strong>如果你正在使用 Android 模拟器，可能就比较麻烦了。</strong></p>
<p>你可以很方便地在 Android Studio 安装的 Android Emulator，它与 VMware Workstation 或 VirtualBox 一样，在较新的版本中都实装了对 Hyper-V 的兼容。</p>
<p>如你需要使用面向手游的 Android 模拟器如 BlueStacks 或者网易 Mumu 模拟器等，你可能就需要进行二选一的操作了。</p>
<p>虽然 BlueStacks 有提供一个兼容 Hyper-V 的测试版本，但是并不稳定。至少在我的尝试下，它并不能正常工作。而在开启了 Hyper-V 之后使用 Mumu 则会发生蓝屏的情况。</p>
<p>如果你只是偶尔使用 WSL，你可以尝试使用夜神模拟器（建议使用国际版且同样内置简体中文）。至少在我的电脑上，它会在运行前自动帮你关闭 Hyper-V，且不会像 Mumu 发生蓝屏 😥。</p>
